# Distribuidora de Vehículos

Este proyecto es una aplicación en Java que permite a los usuarios ingresar sus datos personales y la información de un vehículo. El programa luego muestra todos los datos ingresados en la pantalla.

## Descripción

La aplicación solicita al usuario su nombre y edad, seguidos por los detalles del vehículo, incluyendo la marca, modelo, cilindrada, tipo de combustible y capacidad en pasajeros. Después de ingresar todos los datos, la aplicación muestra un resumen de la información proporcionada.

## Requerimientos Funcionales

1. **Ingreso de datos personales**: 
   - El programa debe solicitar al usuario que ingrese su nombre.
   - El programa debe solicitar al usuario que ingrese su edad.
   
2. **Ingreso de datos del vehículo**:
   - El programa debe solicitar al usuario que ingrese la marca del vehículo.
   - El programa debe solicitar al usuario que ingrese el modelo del vehículo.
   - El programa debe solicitar al usuario que ingrese la cilindrada del vehículo.
   - El programa debe solicitar al usuario que ingrese el tipo de combustible del vehículo.
   - El programa debe solicitar al usuario que ingrese la capacidad en pasajeros del vehículo.

3. **Visualización de datos**:
   - El programa debe mostrar los datos personales ingresados por el usuario.
   - El programa debe mostrar los datos del vehículo ingresados por el usuario.

## Requerimientos No Funcionales

1. **Interfaz de usuario**:
   - La interacción con el usuario se realiza a través de la consola, utilizando la clase `Scanner` de Java para la entrada de datos.

2. **Eficiencia**:
   - El programa debe ser capaz de ejecutar y completar el ingreso y visualización de datos en un tiempo razonable.

3. **Claridad del Código**:
   - El código debe ser claro y comentado adecuadamente para facilitar su comprensión y mantenimiento.
   - Se deben usar nombres de variables descriptivos.

4. **Compatibilidad**:
   - El programa debe ser compatible con cualquier entorno que soporte Java SE 8 o superior.

5. **Manejo de Errores**:
   - El programa debe manejar adecuadamente los errores de entrada del usuario para evitar fallos inesperados.
